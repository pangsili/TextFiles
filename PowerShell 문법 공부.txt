# Azure Active Directory ( Azure AD )

클라우드 기반 ID 및 액세스 관리 서비스이다.

직원이 Microsoft 365, 및 다른 외부 리소스에 엑서스를 할 수 있게 해준다.

# Get / Set -MsolUser

Azure Active Directory 와 관련이 있는것인가? 없는 것인가? 그게 헷갈린다.

MsolUser : Azure Active Directory 에 있는 사용자

Get-MsolUser : Azure Active Directory 에 있는 사용자를 불러온다.

Set-MsolUser : Axure Active Directory 에 있는 사용자를 수정한다.

# SMB ( Server Message Block )

디렉터리 및 주변 장치들을 공유하는데 사용되는 메시지 형식


# Resource

프로그램들이 활용 할 수 있는 데이터나 루틴들을 가리키며 또,

커다란 시스템의 일부를 이루는 하드웨어, 소프트웨어, 또는 데이터의 한 구성요소를 말한다.


# Context

일반적으로는 문맥, 맥락이라는 뜻을 가지고 있다.

런타임 시에 생성되는 환경설정 정보이다.


# Blob

Binary Large Object 의 약자이다.

바이너리 형태로 큰 객체를 저장할 것이라고 추측이 가능한데,

이 큰 객체라 함은 주로 이미지, 비디오, 사운드 등과 같은 멀티미디어 객체들을 주로 가리킨다.


# Binary

0 과 1 을 이용한 2진수 데이터만으로 이루어진 파일로, 사람이 직접 읽을 수 없다.

대표저긴 확장자는 exe, dll, zip, rar, mp3, mpg, jpg, png 등이 있다.


# SharePoint ( SP )

조직에서 웹 사이트를 만들 수 있도록 설계된 브라우저 기반 콘텐츠 관리 시스템 및 응용 프로그램 플랫폼이다.

문서와 파일을 저장, 구성 및 공유하는 안전한 공동 작업 공간 또는 정보 포탈 역할을 한다.


# SharePoine Online ( SPO )

SharePoint 의 Microsoft Cloud 버전이다.


# Cloud

인터넷을 통해 액세스 할 수 있는 서버와 이러한 서버에서 작동하는 소프트웨어와 데이터베이스를 의미한다.


# Document

웹 페이지 그 자체를 의미한다.

웹 페이지에 존재하는 HTML 요소에 접근하고자 할 때는 반드시 Document 객체부터 시작해야 한다.


# library

소프트웨어 개발 시 사용되는 프로그램의 구성요소로, 

공통으로 사용될 수 있는 특정한 기능들을 모듈화한 것이다.

모든 기능들을 손코딩으로 작성하는 방법은 시간 등이 너무 많이 소요되기 때문에 라이브러리 기능을 사용한다.


# Module

소프트웨어 설계에서 기능단위로 분해하고 추상화 되어 재사용 및 공유 가능한 수준으로 만들어진 단위이다.

그 자체로 하나의 기능을 수행할 수 있는 독립된 실체로,

본테에서 분리되어 작은 부분으로 유기적으로 구성되어 있다가, 필요할 때에 본체에 합류하여

그 기능을 수행 할 수 있다.


# Module 화

거대한 문제를 작은 조각의 문제로 나누어 다루기 쉽도록 하는 과정


# CSV ( Comma-Separated Values ) 파일

몇 가지 필드를 쉼표 ( , ) 로 구분한 텍스트 데이터 및 텍스트 파일이다.

각 라인의 칼럼들이 콤마로 분리된 텍스트 파일 포맷이다.


# Format

일정한 모양 또는 형식

데이터를 기억, 인쇄하기 위해 설정하는 일정한 형식


# Text

사람들이 읽을 수 있는 일련의 글자와, 컴퓨터가 읽을 수 있는 형식으로 바꿀 수 있는 단어들을 말한다.

그래픽 이미지 등이나 바이너리 코드 등 글자가 아닌 데이터와 구분하기 위해 사용되는 용어이다.
