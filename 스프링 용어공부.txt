프레임워크란? (프레임워크 vs 라이브러리)

프레임워크는 뼈대나 기반 구조를 뜻하고, 제어의 역전(IoC) 개념이 적용된 대표적인 기술이다.

소프트웨어에서의 프레임워크는 "소프트웨어의 특정 문제를 해결하기 위해서 상호 협력하는

	클래스와 인터페이스의 집합" 이라고 할 수 있으며, 완성된 어플리케이션이 아닌 프로그래머가

	완성시키는 작업을 해야 합니다.

라이브러리는 단순 활용가능한 도구들의 집합을 말한다.

즉 개발자가 만든 클래스에서 호출하여 사용, 클래스들의 나열로 필요한 클래스를 불러서 사용하는 방식을 취함.

★ 제어의 역전(IoC)

하나의 설계원칙으로, 디자인 패턴이라고도 한다.

프로그래머가 직접 객체의 생성과 소멸 객체간 관계같은 객체의 제어를 수행하는 것이 아니라,

여러 프레임워크, 컨테이너에서 제어를 수행하는 것이다.

	※ 스프링 컨테이너

	자바 객체의 생명 주기를 관리하며, 생성된 자바 객체들에게 추가적인 기능을 제공하는 역할

	여기서 말하는 자바 객체가 스프링에서는 빈(Bean) 이라고 한다.

	스프링 컨테이너에는 IoC 와 DI 의 원리가 적용된다.

확장이 가능하고 모듈화된 프로그램을 구성하는 "느슨한 결합" 을 달성하기 위해 다양한 종류의

컨트롤을 반전하는 것을 의미한다.

이러한 제어의 역전은 클래스 간의 결합을 느슨하게 설계하여 테스트가 가능하고,

유지보수가 용이하게 만드는데 도움이 된다.

★의존성 주입(DI)

객체 지향에서 다른객체를 사용하는 것을 다른 객체에 의존한다 라고 표현한다.

객체를 사용할 때는 객체 안에서 다른 객체를 new 하여 직접 생성하는 방식이 있고,

객체 외부에서 new로 생성되어 객체 내부로 주입하여 사용하는 방식이 있는데,

의존성 주입은 이중 후자에 해당한다.

★인터페이스

자바에서 인터페이스란 클래스들이 필수로 구현해야 하는 추상 자료형이다.

쉽게 말해서 객체의 사용방법을 가이드라인 하는 것이다.

자바의 인터페이스는 추상 메서드와 상수로만 이루어져 있다.

구현된 코드가 없기 때문에 당연히 인터페이스로 인스턴스도 사용할 수 없다.

--인터페이스의 특징--

-다중 상속 가능

인터페이스는 껍데기만 존재하여 클래스 상속 시 발생하는 모호함이 없다. 고로 다중 상속이 가능하다.

	※ 상속

	상속을 사용하는 이유는 결국 더 좋은(빠르고, 유지가 쉽고, 중복이 적고, 통일성이 있는)

	프로그램을 만들기 위해서이다.

	자식(클래스)이 상속받고 싶은 부모(클래스)를 선택해서 물려받는다.

	이 때 상속받는 클래스를 자식 클래스, 하위 클래스, 서브 클래스 등으로 부른다.

	상속을 해주는 클래스를 부모 클래스, 상위 클래스, 슈퍼 클래스 등으로 부른다.

	자바에서는 자식 클래스가 여러 부모롤부터 다중 상속을 받는 것은 불가능하지만,

	부모 클래스는 여러 개의 자식 클래스에게 상속이 가능하다.

	부모 클래스는 자식 클래스에서 정의한 메소드나 필드를 사용하지 못한다.

	자식 클래스가 더 다양한 기능이 있게 되므로 자식 클래스로 인스턴스 생성이 효율적이다.

-추상 메서드와 상수만 사용 가능

인터페이스에서는 구현 소스를 생성할 수 없다. 고로 상수와 추상 메서드만 가질 수 있다.

-생성자 사용 불가

인터페이스 객체가 아니므로 생성자를 사용할 수 없다.

-메서드 오버라이딩 필수

자식 클래스는 부모 인터페이스의 추상 메서드를 모두 오버라이딩 해야 한다.

	※ 오버라이딩

	상속받은 부모 클래스의 메서드를 이름은 그대로 두고 재정의하여 사용하는 것

☆ 추상 자료형

기능의 구현 부분을 나타내지 않고 순수한 기능이 무엇인지 나열한 것

예) 선풍기 사용 설명서

	사용 설명서에 정지, 미풍, 약풍 등이 있지만, 그것을 눌렀을 때, 

	내부회로에서 무슨일이 일어나는지에 대한것은 설명하지 않는다.

사용자는 내부 구현이 어떻게 되어 있는 알 필요없이, 추상 자료형을 보고 목적에 맞게 가져다 쓰면 된다.

구현자 입장에서는 구현을 위한 기능과 데이터 집합을 나열하는 것이다.

추상 자료형은 정해진 것이 아니고, 자신의 구현 목적에 따라서 얼마든지 달라질 수 있다.

☆인스턴스

인스턴스란, 클래스의 정의를 통해 만들어진 객체를 의미한다.
